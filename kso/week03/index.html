项目概述

本项目实现简单图形和复合图形的周长和面积计算。

项目基于 CMake 构建，下表列出了项目包含的 CMake 目标：

目标 CMake 目标名 依赖项 类型 说明
主要目标 week03 静态库 包含核心功能。计算图形的面积和周长。
单元测试 Test_SimpleShape week03 可执行文件 负责测试项目功能。
API 文档* Doc 实用工具 生成 API 文档，并用浏览器打开。
代码覆盖* CodeCov Test_SimpleShape 实用工具 生成代码覆盖率报告。
内存检测* MemCheck Test_SimpleShape 实用工具 检查内存泄漏。

*：默认不生成，需要修改 CMakeCache.txt 中的值来启用

项目可以在 Linux 、Windows 等平台编译运行。

测试核心模块

前提

  • C++工具链（MSVC、MinGW、gcc等均可）
  • Boost 库头文件

单元测试依赖 Boost 的头文件（不需要编译 Boost ），如有必要，请设置 BOOST_ROOT 变量到 Boost 根目录以便 CMake 找到 Boost。

构建和运行

在 week03 目录下执行以下命令以完成构建：

mkdir build
cd build
cmake ..
cmake --build .

要运行单元测试，执行：

ctest

如果使用的是 MSVC，单元测试自带内存泄漏检测功能。如果使用的是 MinGW 或 gcc，请参考下文用 valgrind 执行内存泄漏检测。

生成 API 文档

前提

  • doxygen

构建和运行

打开 week03/build/CMakeCache.txt，将 BUILD_DOC 设置为 ON。

在 week03/build 目录下执行以下命令：

cmake ..
cmake --build . --target Doc

即可生成并打开API文档。

代码覆盖

前提

  • C++工具链（MinGW或gcc）
  • gcovr

gcovr 是python包，可以用 pip install gcovr 安装。

代码覆盖依赖gcov，故不支持MSVC工具链。

构建和运行

打开 week03/build/CMakeCache.txt，将 BUILD_COVERAGE 设置为 ON。

在 week03/build 目录下执行以下命令：

cmake ..
cmake --build . --target CodeCov

即可执行代码覆盖检测。

内存检测（Valgrind）

前提

  • valgrind

Valgrind 只支持 Linux 平台。如果要在 Windows 平台检测内存泄漏，请参考测试核心模块一节运行单元测试，Boost 在此期间会自动检测内存泄漏。

构建和运行

打开 week03/build/CMakeCache.txt，将 BUILD_MEMCHECK 设置为 ON。

在 week03/build 目录下执行以下命令：

cmake ..
cmake --build . --target MemCheck

即可执行 MemCheck。

也可以执行ctest来运行测试和 MemCheck。
