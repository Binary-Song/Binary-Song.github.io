Week 4 作业

1. 项目概述

本项目实现矩阵类。支持矩阵相关运算。

2. 项目特点

内存安全

迭代器使用智能指针。在解引用前会自动检查是否指向有效内存，无效会报错。不用担心野迭代器会造成“未定义行为”。

懒拷贝

矩阵包含指向数据内存块的指针。拷贝时首先浅拷贝，如果后面修改了，则再执行深拷贝； 如果从未修改，则省下一次深拷贝的开销。

高效变换

通过创建视图，以下操作能在 O(1) 时间内完成：

  • 转置（transpose()）
  • 变形（reshape()）

Cache 友好

使用连续内存存放矩阵，对 Cache 友好。

3. API

矩阵支持大量操作，例如：

逐个元素的计算：

  • 四则：+ - * / %
  • 逻辑：&& || !
  • 位： & | ^
  • 比较： > < >= <= == !=
  • 复合赋值： += -= *= /= %= &= |= ^=

线性代数：

  • 矩阵点乘：dot()

矩阵修改：

  • 删除行、列：eraseRow() eraseCol()

API文档可以在此查看。 点此进入

4. 构建指南

项目基于 CMake 构建，可以在 Linux 、Windows 等平台编译运行。

单元测试

前提

  • C++工具链（MSVC、MinGW、gcc等均可）
  • Boost 库头文件

单元测试依赖 Boost 的头文件（不需要编译 Boost ），如有必要，请设置 BOOST_ROOT 变量到 Boost 根目录以便 CMake 找到 Boost。

构建和运行

在 week04 目录下执行以下命令以完成构建：

mkdir build
cd build
cmake ..
cmake --build .

要运行单元测试，执行：

ctest

如果使用的是 MSVC，单元测试自带内存泄漏检测功能。如果使用的是 MinGW 或 gcc，请参考下文用 valgrind 执行内存泄漏检测。

API 文档

前提

  • doxygen

构建和运行

打开 week04/build/CMakeCache.txt，将 BUILD_DOC 设置为 ON。

在 week04/build 目录下执行以下命令：

cmake ..
cmake --build . --target Doc

即可生成并打开API文档。

代码覆盖

前提

  • C++工具链（MinGW或gcc）
  • gcovr

gcovr 是python包，可以用 pip install gcovr 安装。

代码覆盖依赖gcov，故不支持MSVC工具链。

构建和运行

打开 week04/build/CMakeCache.txt，将 BUILD_COVERAGE 设置为 ON。

在 week04/build 目录下执行以下命令：

cmake ..
cmake --build . --target CodeCov

即可执行代码覆盖检测。

输出

在控制台会有字符形式的输出

在 build/CodeCov/ 可以找到 html 格式的输出（index.html）

内存检测（Valgrind）

前提

  • valgrind

Valgrind 只支持 Linux 平台。如果要在 Windows 平台检测内存泄漏，请参考测试核心模块一节运行单元测试，Boost 在此期间会自动检测内存泄漏。

构建和运行

打开 week04/build/CMakeCache.txt，将 BUILD_MEMCHECK 设置为 ON。

在 week04/build 目录下执行以下命令：

cmake ..
cmake --build . --target MemCheck

即可执行 MemCheck。

也可以执行ctest来运行测试和 MemCheck。
